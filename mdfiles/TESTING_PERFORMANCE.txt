PERFORMANCE TESTING & VERIFICATION GUIDE
========================================

✅ STEP 1: CREATE DATABASE INDEXES (Required)
==============================================

Run the optimization script:

```bash
docker-compose exec postgres psql -U solar_user -d solar_sharing -f create_indexes.sql
```

Output should show:
  CREATE INDEX
  CREATE INDEX
  (6 indexes total)


✅ STEP 2: VERIFY FRONTEND CACHING WORKS
=========================================

In browser console, try this:

1st request (should be slow):
```javascript

  requiredKwh: 10,
  maxPrice: 10,
  renewable: false
}).then(data => console.log('Result:', data.matches.length, 'matches'))
```

  Result: 15 matches

2nd request immediately after (should be instant):
```javascript
matchingApi.findSellers({
  requiredKwh: 10,
  maxPrice: 10,
  renewable: false
```

Observe in console:

  Result: 15 matches

If you see [CACHE HIT], caching is working! ✅

✅ STEP 4: TEST TIMEOUT FALLBACK
================================

Simulate slow ML service:

1. Stop ML service:
   docker-compose stop ml-service

2. In frontend, fetch matches:
   - Should wait 8 seconds
   - Should return fallback results (just sorted by rating)
   - Should NOT show timeout error

Expected behavior:
  [ML_TIMEOUT] ML service slow, returning best fallback
  ✅ Still get results!

3. Restart ML service:
   docker-compose up -d ml-service

Next request should get optimal ML matches again.


✅ STEP 5: LOAD TEST
====================

Simulate multiple concurrent requests:

```bash
# Terminal 1: Watch logs
docker-compose logs -f backend | grep MATCHING

for i in {1..5}; do
```

  [MATCH_COMPLETE] Found 5 matches in 3095ms

✅ STEP 6: MEMORY MONITORING
============================

```bash
# In another terminal: Repeatedly fetch matches
for i in {1..50}; do
```

Memory should:

  ❌ Should NOT keep increasing after 5 min

✅ STEP 7: RESPONSE TIME ANALYSIS
=================================

Use curl with timing:

```bash
curl -w '@curl-format.txt' \
  -X POST http://localhost:3000/api/v1/matching/find-sellers \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{"requiredKwh": 10, "maxPrice": 10}' \
  http://localhost:3000/api/v1/matching/find-sellers
```

Create curl-format.txt:
```
    time_namelookup:  %{time_namelookup}
       time_connect:  %{time_connect}
    time_appconnect:  %{time_appconnect}
   time_pretransfer:  %{time_pretransfer}
      time_redirect:  %{time_redirect}
 time_starttransfer:  %{time_starttransfer}
                    ----------
         time_total:  %{time_total}
```

Expected times:
  - time_connect: <50ms
  - time_starttransfer: 1500-3000ms (DB + ML)
  - time_total: 1500-3000ms
