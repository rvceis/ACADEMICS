     - Better error handling with fallback

     - Cleaner code

API ENDPOINTS USED
==================

For exports:
  GET /api/metadata/{recordId}/data
  - Fetches all data rows
  - Query params: per_page, filter_field, etc.
  - Returns: { data: [...], total_rows: ... }

For record values:
  GET /metadata/{recordId}
  - Fetches single record
  - Response: { values: {...}, ... }

For schema fields:
  GET /schemas/{schemaId}
  - Fetches schema structure
  - Response: { fields: [...], ... }
